# Security Policy

## Supported Projects

Security support applies only to repositories that are:
- Public, and
- Actively maintained, and
- Explicitly marked as production-ready in their README

Experimental, archived, or internal repositories are not covered unless stated otherwise.

---

## Reporting a Vulnerability

If you believe you have found a security vulnerability in a public repository
maintained by Multinest Media, please report it responsibly.

**Please do not** open a public GitHub issue for security-related findings.

---

## Responsible Disclosure

We ask that you:
- Allow reasonable time for investigation and remediation
- Avoid public disclosure until the issue has been addressed or acknowledged
- Act in good faith and avoid exploiting the vulnerability

Multinest Media commits to:
- Acknowledging receipt of reports in a reasonable timeframe
- Assessing reported issues based on severity and impact
- Taking appropriate remediation steps where warranted

---

## Scope & Safe Harbor

This policy applies only to vulnerabilities discovered through:
- Passive analysis
- Normal use of the software
- Non-destructive testing

Activities that degrade service availability, access private data,
or violate applicable laws are outside the scope of this policy.

---

## Security Updates

When applicable, security fixes will be documented in:
- Release notes
- Changelogs
- Repository documentation

Not all repositories receive ongoing security updates.
Refer to individual repository documentation for maintenance expectations.
